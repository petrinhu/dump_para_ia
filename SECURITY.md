# Política de Segurança

## Reportar vulnerabilidade

Se encontrar uma vulnerabilidade de segurança:

1. Não abra issue pública
2. Envie email para [insira seu email ou contato] com:
   - Descrição detalhada
   - Passos para reproduzir
   - Impacto potencial
   - Sugestão de correção (opcional)

## Escopo

- Vulnerabilidades que permitam execução arbitrária de código
- Exposição de dados sensíveis
- Falhas de integridade no dump gerado

## Processo

- Reconhecimento em até 48h
- Análise e correção em até 30 dias (prioridade alta)
- Crédito ao relator na CHANGELOG/RELEASE (se desejar)

Agradecemos por ajudar a manter o projeto seguro.
